Опыт работы с программами, средами, языками

  • Продвинутый пользователь программ MS Office: Word, Excel, Visio, Project.
  • Работал с продуктами Atlassian: Jira, Confluence, Bitbucket.
  • В качестве аналогов продуктов Atlassian в различное время использовал Gitlab, Plane, YouTrack.
  • Для документации API применял Swagger. Документировал только REST, с SOAP знаком, но в работе не сталкивался.

С Agile знаком, но заострять внимание на его использовании в работе смысла не вижу - сейчас ни одна IT-компания не выживет без его принципов.

На чем пробовал документировать:

Название Причина выбора, оценка
Markdown Не нуждается в комментариях. Вариант навека как для простых README, так и для крупных баз знаний. В качестве движка имею опыт работы с mkdocs.
R Markdown Выбрал его самостоятельно на один из проектов, поскольку он является неплохой альтернативой Word для создания документов. Очень понравилось внедрять чанки языка R для упрощения подготовки документа.
LaTeX Испытывал себя в области подготовки научных публикаций, вдобавок он интегрирован в R Markdown. Самой удобной частью считаю генерацию формул, в остальном он не очень гибок.

В качестве генерации баз знаний довелось использовать Doxygen. Вряд ли еще пригодится, но мало ли.

Для разработки диаграмм использовал как простой UML, так и интегрировал PlantUML. Работал с нотацией BPMN в процессе создания схем.
